Static Private onUploadDisabled

  • onUploadDisabled(): Promise<void>
    • Defined in core/glean.ts:101

    Handles the changing of state from upload enabled to disabled.

    Should only be called when the state actually changes.

    A deletion_request ping is sent, all pending metrics, events and queued pings are cleared, and the client_id is set to KNOWN_CLIENT_ID. Afterward, the upload_enabled flag is set to false.

    Returns Promise<void>

Static setUploadEnabled

  • setUploadEnabled(flag: boolean): void
    • Defined in core/glean.ts:362

    Sets whether upload is enabled or not.

    When uploading is disabled, metrics aren't recorded at all and no data is uploaded.

    When disabling, all pending metrics, events and queued pings are cleared.

    When enabling, the core Glean metrics are recreated.

    If the value of this flag is not actually changed, this is a no-op.

    Parameters

    • flag: boolean

      When true, enable metric collection.

    Returns void

    • Defined in core/glean.ts:576

    Test-only API**

    Resets the Glean singleton to its initial state and re-initializes it.

    Note: There is no way to only allow this function to be called in test mode, because this is the function that puts Glean in test mode by setting Context.testing to true.

    Parameters

      The application ID (will be sanitized during initialization).

    • uploadEnabled: boolean = true

      Determines whether telemetry is enabled. If disabled, all persisted metrics, events and queued pings (except first_run_date) are cleared. Default to true.

    • Optional config: ConfigurationInterface

      Glean configuration options.

    • clearStores: boolean = true

      Whether or not to clear the events, metrics and pings databases on reset.

    Returns Promise<void>
